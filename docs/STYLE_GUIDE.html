Semantic Color Usage

Color Use For Never Use For
Blue / Primary Links, CTAs, info badges, "In X days" deadlines Errors, success states
Green / Success Completed, approved, on track Warnings, upcoming deadlines
Orange / Warning At risk, pending, "Tomorrow" deadlines Success states, errors
Red / Danger Overdue, urgent, "Today" deadlines, errors Success, neutral info
Amber / Holiday Holiday-related components only Status indicators, badges

Usage Rule

Use DM Mono for any data that changes or is machine-generated: progress counts (8/14), dates, timestamps, percentages. Use DM Sans for everything else.

Usage Rule

Cards start with no shadow and gain shadow-md on hover. This creates subtle depth without visual noise at rest.

CSS Variables

Copy this to your stylesheet root.

/* Backgrounds */ --bg-primary: #FAFAFA; --bg-card: #FFFFFF; --bg-subtle: #F5F5F5; --bg-accent: #0A0A0A; /* Text */ --text-primary: #0A0A0A; --text-secondary: #6B6B6B; --text-tertiary: #9A9A9A; --text-inverse: #FFFFFF; /* Semantic */ --accent-primary: #2563EB; --accent-success: #059669; --accent-warning: #D97706; --accent-danger: #DC2626; /* Borders */ --border-light: #E5E5E5; --border-medium: #D4D4D4; /* Radius */ --radius-sm: 8px; --radius-md: 12px; --radius-lg: 16px; /* Shadows */ --shadow-sm: 0 1px 2px rgba(0,0,0,0.04); --shadow-md: 0 4px 12px rgba(0,0,0,0.06); --shadow-lg: 0 8px 24px rgba(0,0,0,0.08); /* Typography */ --font-sans: 'DM Sans', -apple-system, sans-serif; --font-mono: 'DM Mono', monospace;
